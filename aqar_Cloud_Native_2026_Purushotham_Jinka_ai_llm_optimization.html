Database Evolution: Partitioning and Indexing for Cloud-Native Systems

Video size:

Abstract

The rapid adoption of cloud computing, growing at 39% year-over-year across enterprises, is reshaping how modern databases are designed and operated. This session explores how partitioning and indexing strategies have evolved from rigid, on-premises models to intelligent, adaptive mechanisms built for cloud-native architectures. We’ll examine how modern partitioning approaches dynamically adapt to workload patterns, improving query response times by 47% while reducing administrative overhead by 60%. We will also discuss the transformation of indexing from static B-tree structures to adaptive, multi-dimensional models that reduce storage overhead by 28% and boost query performance by up to 35%. Using benchmarks from frameworks such as OLTP-Bench and iTuned, the session highlights how cloud-native database platforms sustain >99.95% availability, scale elastically to support 10× workload surges within minutes, and process over 10 million transactions per second while lowering operational costs by up to 41% and reducing mean-time-to-resolution by 43% in large deployments. The talk also explores the growing role of machine learning in next-generation database engines. Learned indexes can shrink index size by 10×, adaptive query processors reduce execution time by 45%, and ML-driven workload managers achieve up to 90% prediction accuracy. These capabilities enable self-optimizing, resilient, and cost-efficient database systems while improving developer productivity by up to 50% and reducing DBA workload by 60%.
