Bridging the Security Tool Gap for Go

Video size:

Abstract

This talk introduces OpenRefactory’s Intelligent Code Repair (iCR) tool which provides support for Go along with Java and Python. iCR finds bugs that other tools miss, finds bugs with low false warnings, and synthesizes fixes. iCR fills in the gap in Go language’s tool support.

Summary

  • Munawar Hafiz: Over 150 security vulnerabilities are reported every week. Current static analysis tools fall short of detecting these bugs in time.   • ICR is a suite of Docker containers that control access to your code repositories. It analyzes your projects, and allows you to review and accept the results. Using ICR is simple and effective in finding critical security vulnerabilities.
  • Sonar cloud identifies 33 security bugs, six reliability bugs, and 257 maintainability bugs in the code. ICR was able to synthesize a fix for one of the bugs. We are still searching for a tool that can identify the critical security problems.
