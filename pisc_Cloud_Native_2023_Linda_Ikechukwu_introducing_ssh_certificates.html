A better way to SSH: Introducing SSH certificates

Video size:

Abstract

SSH certificates are the most secure way of granting SSH access. However, they’re not widely used. In this talk, you’ll learn of the shortcomings of SSH keys, what SSH certificates are, how SSH certificates combat the shortcomings of SSH keys, and how you can start using SSH certificates right away!

Summary

  • Most popular way of authenticating SSH access is using SSH keys. Traditional SSH keys usage has private keys stored on digs. These can often be stolen using malware or red if a team member's device is compromised. Is there a better solution? And yes, there is, and that's SSH certificates.
  • Linda Ikechukwu is a developer advocate at Smallstep Labs. She talks about how SSH certificates work and why they are superior to traditional SSH keys authentication. Also shares some links that will help you learn or set up SSH certificates in your own infrastructure.
  • SSh keys are not so bad as I mentioned. When they are properly secured and managed, they could be the simplest solution for a small of, say, ten to 20 devices. But they are just not scalable or robust for larger organizations.
  • Ssh key authentication works kudos to the magic of asymmetric cryptography. When you ssh into a server using your public key, the SsH server challenges your client to sign something with your private key. If it matches, then you can successfully log in.
  • The process of onboarding SSH keys is time consuming and not scalable for organizations with hundreds to thousands of servers. The next reason why SSH keys are really bad for you is that it encourages unsafe user behavior and bad security practices. The task of gathering and shipping and maintaining SSH keys always remains messy.
  • The truth is you can't tamessh keys. They just have a rudimentary fundamental design flaw. And so the answer would be to just use SSH certificates. Here's what an SSH key certificate looks like when decoded.
  • To get started using SSH certificates based authentication, the only additional element you need to introduce to your infrastructure is an SSH authority. Now in this case you would configure clients and hosts to trust certificates issued by your CA. Some tools make this process automated.
  • Using SSH certificate authentication is far superior to traditional SSH key authentication. Both clients and hosts can be sure they are connecting to infrastructure that actually belongs to their organization. Another great thing about using SSH certificates is that they are easier to maintain and manage.
  • SSH certificates authentication also allows for extensibility. You can issue SSH certificates via a variety of authentication mechanisms to facilitate better usability. We're convinced that SSH certificates are the best way to do SSH. I really hope that you consider using SSH certificates in your infrastructure and in your organization.
