Your AI Has Too Much Access: Deterministic Authorization for Coding Agents

Video size:

Abstract

Your coding agent needs better permissions, and so does the code it writes.

AI Agents without explicit boundaries can take actions outside their mandate. A common approach to preventing this is letting the agent decide for itself what’s safe. In fact, Anthropic’s analysis of Claude Code’s auto-permissions mode reports a 17% false-negative rate on dangerous actions and is upfront about the tradeoff. This is a big limitation in enterprise because Agents are making probabilistic judgments but authorization requires deterministic answers.

This session walks through how you can add explicit, enforceable permission boundaries to your coding agent using SpiceBox, an open-source tool built on SpiceDB. SpiceBox is built on the Google Zanzibar model of authorization which uses Relationship-Based Access Control (ReBAC). This talk explains how this tool secures your agent, and why traditional models such as Role-Based Access Control break down when you need to scope permissions by path, command prefix, or domain. There’s a deep-dive into how the tool can prevent common threat vectors such as prompt injection, data exfiltration, Multi-tenant MCP tool access, and more.

The talk includes a live demo that will show scoping a coding agent to not write outside its allowed paths, and see what network isolation actually looks like in practice. We’ll also show how an agent can request more access and a human can grant it cryptographically. Learnings from this talk could prevent a big data breach!
