Post-Quantum Cryptography: Securing Multi-Cloud APIs Against Quantum Threats

Video size:

Abstract

The emergence of cryptographically relevant quantum computers poses an existential threat to cloud-native API architectures that depend on RSA and Elliptic Curve Cryptography for securing data-in-transit and establishing trust. Quantum algorithms capable of solving discrete logarithm and integer factorization problems will render current public-key infrastructure obsolete, exposing organizations to harvest-now-decrypt-later attacks where adversaries collect encrypted data today for future decryption. This session shares practical lessons from assessing and mitigating quantum risk across distributed API ecosystems on heterogeneous cloud platforms. A hands-on risk assessment framework will be presented covering cryptographic asset discovery, data classification based on confidentiality timelines, and API criticality evaluation. The session examines standardized post-quantum algorithms, including ML-KEM (Kyber) for key encapsulation and ML-DSA (Dilithium) for digital signatures, with honest comparisons of their security properties, performance trade-offs, and real-world deployment considerations against classical alternatives. Mitigation centers on hybrid cryptographic implementations combining classical and post-quantum algorithms within TLS, providing defense-in-depth during the transition period while maintaining cryptographic agility. Implementation challenges are examined through concrete examples, including public key size increases (3-10x larger than ECC), latency overhead (10-30%), and client ecosystem heterogeneity spanning modern cloud runtimes to legacy systems. Attendees will leave with practical guidance on building phased migration roadmaps, prioritizing high-risk components, and navigating the 24-36 month timeline for deploying quantum-resistant infrastructure across multi-cloud environments.
