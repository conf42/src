agentctl: A CLI for Managing AI Agents and MCP Servers Like Infrastructure

Video size:

Abstract

You have kubectl for pods, docker for containers, and helm for charts. But what do you use to register, health-check, start, stop, and discover the growing fleet of AI agents and MCP servers running on your machine and in your cluster? agentctl is a zero-dependency Go CLI that treats AI agents and Model Context Protocol servers as first-class managed services. It provides the same operational primitives you expect from infrastructure tooling — registration, lifecycle management, health checks, tagging, auto-discovery, and a live TUI dashboard — applied to the new layer of AI services that’s quietly becoming load-bearing in cloud-native stacks. This talk is a live walkthrough of building and using agentctl: Part 1 — The problem (5 min) You’re running Ollama locally, three MCP servers across Cursor and Claude Desktop, a LiteLLM proxy, and two custom agents. They’re all managed differently — some are Docker containers, some are Node processes, some are just “I left a terminal tab open.” There’s no unified view. Health checking is manual. When one crashes overnight you find out the hard way. Part 2 — Registration and lifecycle (8 min) We register agents and MCP servers with typed metadata: transport (stdio, SSE, HTTP), confidence thresholds, endpoints, commands, and tags. agentctl agent start launches the process, captures the PID, streams logs to ~/.agentctl/logs/, and tracks status. agentctl mcp health probes endpoints or checks PIDs for stdio transports. Everything persists to a single JSON file — no database, no daemon. Part 3 — Auto-discovery (7 min) agentctl discover scans three sources in parallel: running processes (matching ~30 known signatures including Ollama, LM Studio, CrewAI, LangServe, and any node process with “mcp” in its name), well-known ports (11434, 1234, 4000, 3000, etc.), and MCP config files from Claude Desktop, Cursor, and VS Code. New services are registered automatically. Re-runs are idempotent. Part 4 — The dashboard (5 min) agentctl dashboard renders a live terminal UI refreshing every 3 seconds — agents, MCP servers, recent API calls, health status, confidence scores, all in one view. It’s the htop for your AI service layer. Part 5 — Confidence-gated decisions (5 min) Each agent carries a confidence score (0–1) that maps to a tiered decision framework: green (≥0.85, auto-proceed), yellow (0.60–0.85, notify-proceed), red (<0.60, escalate). We show how this integrates with the broader incident automation platform to gate which actions an agent can take autonomously. You’ll leave with: - A working pattern for managing AI agents as infrastructure primitives - The auto-discovery approach for finding agents and MCP servers across your environment - A confidence-gating framework applicable to any autonomous agent system - The tool itself — pure Go, zero dependencies, install with go install
