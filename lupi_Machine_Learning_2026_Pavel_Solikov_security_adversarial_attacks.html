Stop Failures Before They Happen: Gradient Boosting for Perfect Flows

Video size:

Abstract

Predicting failures before they happen saves time and resources! Running Prefect workflows for JPMC’s X-Margin team is time- and resource-intensive, and failures can go unnoticed until after the calculation. Pavel will explain how training a gradient boosting (XGBoost) model on workflow metadata—inputs, run parameters, and code revisions—enables the system to predict failures in advance. Developers are then alerted instantly, allowing them to fix problems early, saving time, reducing costs, and improving reliability.
