The 5 developers’ principles that helped me as a new father

Video size:

Abstract

You won’t believe #7!

Are you tired? Is your child keeping you up at night? This talk is for you! SRP, The Boy Scout Rule, KISS, YAGNI, Tell, Don’t Ask!

In this talk, we will go through real-life examples that will help us to better understand how to write better code. I will share with you how I used these principles as a new father so I can get more sleep and peace of mind.

Summary

  • Jamaica real time feedback into the behavior of your distributed systems. observing changes exceptions errors in real time allows you to respond instantly to get things working again. The five developer principles that help me as a new father.
  • The first one is to fight coupling as hard as you can. The second is to keep your code as simple as you could and also be kind. We will talk about each of these principles.
  • The idea behind the SRP is that every class, module or function in a program should have one responsibility or purpose. Let's start with some code examples to illustrate it better.
  • Next one is the boy scout rule. Leave your code better than you found it. The more complex something is, the more ways it fails. Keep it simple stupid, not saying stupid.
  • Always implement things when you actually need them, never when you just foresee that you may need them. Unless you already have the infrastructure above because you needed it, there is no need for it. Let's move on to the next one.
  • The tell dont ask principles suggests that it is better to issue a command to perform some operation or logic rather than query its state and then take some action as a result. We move the internal logic of the task inside task class.
