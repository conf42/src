Testy: a minimal testing tool designed for teaching

Video size:

Abstract

Testy is a testing tool that emerged in the “Object-Oriented Programming 2” course at Universidad CatÃ³lica Argentina, where I am the teacher and we learn OOP in dynamic languages like JavaScript.

Testy has 2 main purposes: * Offer a smooth learning experience for students, focused on detailed feedback * Serve as an example of simple, well designed OO open-source codebase, a nice place for first-time contributors

In the presentation, I’ll give an overview of the tool, and a little demonstration of it.

Summary

  • Nahuel Garbezza is a software engineer from Buenos Aires, Argentina. He teaches object oriented programming and test driven development. His new tool, testy, is a minimal testing tool designed for teaching. It doesn't have any dependencies at all.
  • The tool has assertions, a fail fast mode and a way to group test in test suites. It's also multilanguage because I wanted the output to be in other language as well right now, English and Spanish. Two features that I wanted for the tool are beneficial for students.
  • Once you keep adding features and keep adding code, eventually tool might become complex. One visual hint that helped me to control the complexity is this idea of module chart. This is just to have an idea of like a visual idea of how complex your software is.
  • Writing test is as easy as calling a test function that gets imported from our testing module. If there are failures, we will see them here. Unique feature is undetermination if both things are undefined. This is to prevent false positives when writing tests.
