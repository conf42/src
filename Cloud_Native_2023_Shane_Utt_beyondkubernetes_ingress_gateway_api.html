Beyond Kubernetes Ingress (with Gateway API)

Video size:

Abstract

Kubernetes has provided basic controls for ingress traffic to your applications for some years with the Ingress API, but that’s just the beginning of the story. In this talk we’ll discuss Gateway API, the upcoming successor to Ingress with deep and rich controls and configuration options for traffic and operations. We’ll also dive into some of the ongoing and upcoming aspects of the project, including service mesh support and our available tooling and show some practical examples. Attendees should walk away from this talk with a clear idea of what’s coming on the Kubernetes SIG Network roadmap for Gateway API, and should know where to start if they would like to contribute and/or join into the community.
