Keeping Kubernetes secrets secret

Video size:

Abstract

Everyone is talking about the microservices and serverless architecture, and how to deploy them using cluster managers like Kubernetes. But, what about the secrets? Come to this session to see how you can manage your secrets with Vault in Kubernetes; why Kubernetes secrets might be not enough.

Summary

  • This talk is about securing secrets in the Gitops era. A Kubernetes secret is not a secret at all by default. This is the reason for attending this talk to learn how to manage all these secrets.
  • Today we're going to talk about the secrets that needs to manage by our application. A secret is a piece of information that is only known by one person or a few people. There is no 100% secure system. It's about creating layers of protection.
  • What is DevOps? It's that we are able to deliver our applications faster. Basically everything is in git. We are not only having our source code into Git, but also the infrastructure that is required by our application. Everything is automated.
  • The problem is that a Kubernetes secret is not created to be secret at all. Because base 64 is an encoding process and not an encryption process. So anyone can just take this value, as you can see here, do basically four minutes minus the code and get the real value. How to protect these secrets in git.
  • Seal Secrets is a project that uses a public key cryptography for encrypting the secrets. External secrets project is more or less the same thing, but it's a Kubernetes controller running. All these other operations of decrypting happens inside the Kuber netes cluster.
  • In this demo, I'm going to show you how to create a cluster secret store. It's a key value where we are storing in a place secrets in the correct way. In the external case, you can encrypt the secret and store it safely into git.
  • ATCD is where we store the encrypted data, the encrypted secret. But also in the ATCD we're storing the keys for decrypting the secret. If an attacker get access to A TCD, it can get the private key to decrypt the secrets. Possible solutions include using ambient variables and the KMS plugin.
  • Hashicor Vault is the best tools for keeping your secrets secure. It also implements monitoring for attacks. It gives you a lot of functionalities for secret rotations. One of the best things that it has is the dynamic secrets.
