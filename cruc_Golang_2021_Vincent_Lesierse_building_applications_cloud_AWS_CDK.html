Building applications in the Cloud using Go and the AWS CDK

Video size:

Abstract

CDK stands for Cloud Development Kit and it allows developers to define their infrastructure using the code they know and love.

The AWS CDK is now available in preview for Go developers. In this talk we will explore what the AWS CDK is and how it can help you define your Cloud infrastructure.

We will show practical examples of how this CDK can help you build your Go applications in AWS.

Summary

  • Vince Lesa is a solutions architect working for AWS. He shows you how you can build applications using go with AWS CDK. Lesa explains how to deploy these applications to your AWS account.
  • CDK includes the AWS construct library which contains construct representing AWS resources. Level three constructs are like composition of multiple resources across different services. With a single line of code we can actually create a VPC that stands two availability zones.
  • Let's take a look at how we can deploy our own go application using CDk on AWS. We're going to deploy something like a URL shortener which has been written in go and will be deployed as a lambda function. And we can actually look at this output by using this sync command.
  • Using CDK, we can create a simple go application on AWS. We need to create a dynamodb table and a Lambda function. We also need an API gateway to proxy requests to our application. With CDK there are some convenient methods for you to help.
