Make Agents Remember the Right Context at the Right Time

Video size:

Abstract

Agentic systems are rapidly moving from prototypes to productionbut a critical challenge remains unsolved: memory. In real-world deployments, agents dont fail because they cant reasonthey fail because they retrieve the wrong context, forget important signals, or overload themselves with irrelevant information. This talk dives deep into memory architectures for production-grade agent systems, moving beyond naive vector database + RAG patterns. We introduce a practical framework for designing agent memory across three layers: working memory (context window), retrieval memory , and persistent memory and how to orchestrate them effectively. Well explore: Dynamic context management: what to keep, compress, or discard Hybrid retrieval strategies combining semantic, structured signals Well also present a reference architecture and real-world failure scenarios, showing how poor memory design leads to hallucinations, cascading agent errors, and cost blowupsand how to fix them.
