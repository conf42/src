Ducks, Re-ducks, Redux-tools - modular approaches for structuring React/Redux apps

Video size:

Abstract

Redux is a king of the state management in the React world, even though more and more competitors appear. But Redux always had some problems and developers coplained about. We will review what problems developers have with Redux and how the community tried to resolve them in the last several years. We will see evolution of the Redux apps sctructuring - from ducks to the Redux Toolkit. Plan: 1. Redux in 2021? 2. Why good structure for Redux apps is very important 3. What is wrong with the common approaches in Redux apps 4. Ducks in details 5. Re-ducks and how it differs with Ducks 6. What problems Redux Toolkit resolves and how to start to use it. 7. Testing of the duckses

Summary

  • Sergio will talk about modular approaches of structuring react and redux apps. We will check what is dax and reducks, what is redux toolkit, and a little bit about testing of Daxis. What is wrong with common approaches in redux applications?

  • Dax was the first attempt, as for me, to change the rules of the game. After that redux appeared. The difference between DaX and Redux is that Dax's modular approach is nice, but it doesn't work. With Redux, we can easily add tests directly in the DAX folder.
  • Redux toolkit is a logical continuation of this saga with Daxis. They use the same pattern or approaches for improving the structure of the redux application. How do you structure your Redux app? Please share by the way.
