Bye Kubernetes Ingress, Long Life to Gateway API

Video size:

Abstract

Kubernetes’ ecosystems are built to be shared. However, shared infrastructure raises a lot of challenges that could be summarized in having the administration governance and control while allowing flexibility of usage to the users. Thanks Ingress for your amazing job; let’s welcome Gateway API!

Summary

  • Since October 2023, our lovely kubernetes in recipe has been frozen. No features will be added anymore to ingress in favor of the new Gateway API standard. Next a demo regarding how to use Gateway API on Google Cloud platform.
  • Ingress was created to solve a very specific and basic problem. The management of HTTP external access to our workloads running. The main problem here is that ingress configuration reside all in the same file. This led for the poor ingress to have a lack of native advanced usage.
  • Gatewaypi is an open source project managed by the SIG network community. It has reached its very first stability on 31 October 2023. Again, a lot of clap for fans for Gateway API.
  • Gateway API is basically a collection of new customer resources. It is more role oriented and expressive than Ingress API. Many API gateway are already implementing gateway API. In the next demo we will see two Kong Nginx traffic AWS.
  • The hands on is based on Google Cloud platform. We will deploy a simple blue application that will be exposed using ingress. And then we will use Gateway API to do exactly the same thing. See how gateway API can solve a lot of problems that ingress could not do.
  • Let's add some more feature coming from Gateway API using always the developer operator head. For example, let's add a custom meter using always native integration of Gateway API. Let's do a couple of pretty cool example of deploying.
  • Last one I'd like to show you is the blue green deploy which is basically the traffic splitting using a percentage. We are redirecting 50% of the traffic to the blue service and the other 50% to the green one. In this way we can do a sort of gradual release and reduce over the time the traffic.
  • Gateway API is working on a service mesh standard solution. Will Gateway API replace the ingress API? In my opinion, yes. We are enriching basic services from Kubernetes with a lot of features that comes from API.
