Autonomous Multi-Cloud serverless deployment and optimized management

Video size:

Abstract

The dynamic development of Cloud Computing with the introduction of novel Cloud computing models like serverless creates new challenges for Cloud deployment.

This presentation describes how to implement Multi-Cloud native strategies using advanced an open source framework that allows for Cloud-agnostic Multi-Cloud deployment and optimized management of the serverless applications based on flexible monitoring, context aware maximization of the application owner’s utility of the deployed serverless components, and autonomic reconfiguration based on the application’s current execution context.

All stages of the cloud deployment planning and designing process will be shown. Also, the key execution steps will be provided.

Summary

  • The melodic platform is the single universal platform for the automatic multi cloud deployed. It can be deployed to the Azure AWS, Google Cloud, to any OpenStack cloud providers. Everything is done automatically and even more the usage of the resources is optimized. You can consider melodic as your smart and autonomic DevOps operator.
  • How to automatically deploy own application by melodic platform. I will perform deployment of spark based application. We will monitor application metrics and observe reconfiguration process. Here is also possible to run application in simulation mode.
  • Genome is a big data application which performs some calculations and safe results in AWS as free bucket. Genome's performance is managed by Spark. Melodic creates proper number of spark workers as virtual machines considered our requirements. Thanks to measurements of application metrics, melodic makes a decision about creating additional instances with workers or about deleting unnecessary ones.
  • Melodic creates proper instances, remains them or deletes. If you want to have more detailed view, it is possible to see the process view using Kamunda. Grafana is tool for monitoring displaying statistics and metrics. We can use them for monitoring performance of applications deployed by melodic.
  • Melodic is fully open source so you can download the melodic here. It is released under the Mozilla Public License 20 so it can be used and challenges. We are also looking for the volunteers developers, open source developers. If you want to work on the interesting process then please join us.
