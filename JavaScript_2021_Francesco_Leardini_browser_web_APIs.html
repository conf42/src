Wait...can my browser do that?

Video size:

Abstract

We have new powerful instruments to leverage current web apps and provide a richer experience to our users. With the aim of modern web APIs, it is possible to design completely new functionalities and explore unique technological combinations, changing the way we develop and interact with web apps.

We can directly use hardware devices and just through our browser! Let’s explore some of the most exciting APIs and see how we can combine them to unlock new scenarios and give superpowers to our web apps

Summary

  • Today we will learn how modern browser APIs can enhance our web projects. Many of these APIs have the goal to close the feature gap with the native apps. I will show you several demos and of course all the code behind each of the API.
  • The first API we will see is the page visibility API. It provides events we can listen to in order to know when the document changes visibility. We can think practical scenarios of usage of this API. And this is particularly important for mobile users.
  • All the browser API that we will see today, they are absolutely framework agnostic. Therefore you can use in react, vue, js or even vanilla JavaScript. Every section of each API is composed by a short description.
  • Screen wakelock API is a very cool API. Mobile devices typically go on sleep mode or idle after a specific timeout. The screen Wakelock API provides a way to prevent the device from dimming and locking the screen. This is important for web applications that have to provide some content.
  • New file system assets is another very interesting API because it offers a new way to manipulate files in the local file system. This system assets API offers two methods. One is called show open file Picker and another is a show save file picker.
  • Web share is also another cool API. It allows to share an object with a text content, a title and a URL using the device native content sharing capabilities. Chrome recently allowed also on the desktop version the possibility to use this web share API.
  • contact Picker API allows users to select contacts from the device list and use them within the context of a web application. It works mainly with Android browser, opera mobile and of course Chrome for Android. The browser compatibility for this API is not exactly wide yet.
  • Vibration API is our last API for this session. Allows to interact with the hardware of the device. Of course only if the target device allows it to vibrate. Could be another scenario where we can provide some more interesting or entertaining features to our quiz or web app.
