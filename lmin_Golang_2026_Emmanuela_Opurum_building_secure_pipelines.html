Building Secure CI/CD Pipelines with Go-Based Automation

Video size:

Abstract

Modern platform engineering teams are increasingly using Go to build fast, reliable, and cloud-native automation tools for CI/CD and infrastructure management. In this talk, I explore how Go can be used to design secure, scalable CI/CD automation components that support modern DevOps pipelines. Drawing from real-world architecture experience across AWS, Azure, and GCP, I demonstrate how Go-based tooling can improve deployment reliability, enforce governance policies, and reduce operational risk. We will focus on practical engineering patterns rather than theory, including how Go services can be integrated into CI/CD workflows, how to design for observability and resilience, and how to structure automation systems that scale across multi-cloud environments. Attendees will gain insights into: Why Go is well-suited for platform engineering and CI/CD automation How to design secure pipeline automation services using Go Common pitfalls when building infrastructure tooling Architecture patterns for scalable DevOps systems This is a practical, experience-driven session aimed at engineers building real-world delivery platforms.
