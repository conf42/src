Breaking the Monolith: A Comprehensive Guide to Code Splitting Techniques

Video size:

Abstract

Large monoliths have many problems: long builds, a large entry threshold, high cohesion, etc.

One of the most effective ways to solve these problems is to break the monolith. I will tell you what problems may arise during this process, how to solve them and which approach is right for you.

Summary

  • Alexander Gozenka will talk about front end architecture, in particular about splitting the monolith. He will show you six ways how you can split your monolith to reduce complexity of your application. After that we will summarize and find out when which method is best to use.
  • When you use monoliths architecture, you are using a single module that works autonomously independent of other application. As application grows, the benefits of microservices architecture become apparent in terms of both development and maintenance cost. How to recognize when microservices will be cost effective.
  • Next way to split your monolith is using NPM, not package manager. This approach is ideal as a first step before using micro frontends. Last method is using code federation in webpack five.
