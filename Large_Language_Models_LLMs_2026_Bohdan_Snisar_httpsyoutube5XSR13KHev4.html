Voice Agents That Think While They Talk: From STT–LLM–TTS Pipelines to Realtime Parallel Reasoning

Video size:

Abstract

The classic voice-agent pipeline — STT - LLM - TTS — is modular and observable, but every stage waits for the previous one. With a conversation budget under 500 milliseconds, there is almost no room for deep reasoning. Streaming reduces perceived latency but does not solve reasoning depth: the main voice loop is too time-sensitive to block on retrieval, policy checks, or multi-step planning.

This talk builds a production voice pipeline layer by layer from naive chained models through realtime audio transport, VAD, endpointing, and turn detection. Then shows where it hits the wall. We introduce the LTS-VoiceAgent architecture as the turning point: a Dynamic Semantic Trigger starts background reasoning before the user finishes speaking, while a Dual-Role Stream Orchestrator separates a background Thinker (maintaining a delta-state table) from a foreground Speaker that responds from injected state and steers mid-utterance as richer state arrives.

We then extend LTS from one Thinker into a multi-agent sidecar architecture. Standard LLM patterns — intent classification, retrieval, policy checking, tool planning, and evaluation — run as parallel sidecars over the live transcript stream. A reducer merges their outputs, resolves conflicts, and predicts the next action, producing one compact state object for the voice agent.

The user hears one agent. Internally, multiple LLM workflows reason in parallel while the conversation unfolds.
